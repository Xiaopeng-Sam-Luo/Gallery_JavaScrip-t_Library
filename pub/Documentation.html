Documentation of the JavaScript Library Gallery.js

Goal

Select the div in which you want to put the gallery in.

Installation

<script defer type="text/javascript" src='Gallery.js'>

Deveopers can install this library by including the above script ton to the html page

API

selector

Default: /

Type: String

Description: A string that represents the id of certain elemnet(such as div) in the HTML file. It will be used to find the corresponding HTMLElement via document.querySelector

elemnet

Default: /

Type: HTML Element

Description: The search result of document.querySelector(selector). The result represents a div HTML Element, which the developers wants to apply some functionalities from this library.

holders

Default: [ ](empty array)

Type: array

Description: holders is the array that repersetn the order of the list of images that is currently displayed. If the holders array at certain index is null then, it means that position currently has no image. Whenever a new image holder is created, the length of this array would increase one

slides

Default: [ ](empty array)

Type: array

Description: slides is the array that repersetn the order of the list of images that is currently displayed in the image slider. If certain index of slides array is null then, it means that position currently has no image. Whenever a new image slider is created, the length of this array would increase one.

Default: [ ](empty array)

Type: array

If certain index of slides array is null then, it means that position currently has no image. Whenever a new image slider is created, the length of this array would increase one.

Gallery.makeholders

Use to control the number of imageholders that is created.

num

Default:/

Type: number

Description: This function is used to craete multiple image holders at once, the num input paramater represetns the number of image horlders, the developers wants to add to this list. The input parameter num is required otherwise, this funciton would not execute.

Gallery.changesize(index, stylesheet)

Use to overwrite the style of the imageholder at the given index

index

Default:/

Type: number

Description: This input parameter index represerns the index of the holders array.

stylesheet

Default:""

Type: String

Description: This input parameter stylesheet represerns the style that the develpoers wants to overwirte the original style of the image holder.

Gallery.drag & Gallery.order

Give access to HTMLElement object to be draggable and reoderable

drag

Default:false

Type: function

Description: Trigger when develpoers wants to make HTMLElement draggable

order

Default:false

Type: function

Description: Trigger when develpoers wants to make HTMLElement reorderable. Once is triggered

Gallery.insertimages(index, url)

Allow developers to insert images to the corresponding image holder. If there is already a image, the new image will replace the older one.

index

Default:/

Type: number

Description: This input parameter index represerns the index of the holders array.

URL

Default:/

Type: String

Description: This input parameter URL represerns the address of the image which the developers wants to put on the corresponding imageholder.

Gallery.show

name

Default:/

Type: String

Description: This input parameter name represerns the class name which developers wants to name those sliders..

Gallery.pixelRainEffect(height, width, sp, index, method, path)

Create pixel rain effect to the choose image.

height

Default:/

Type: number

Description: This input parameter height represents the height of the HTMLElement of the animated image

width

Default:/

Type: number

Description: This input parameter width represents the width of the HTMLElement of the animated image

sp

Default:0.5

Type: number

Description: This input parameter sp represents the velocity of each moving partical, the larger the faster the velocity. Default veloctiy is 0.5

index

Default:0

Type: number

Description: can only be chose from 0 or 1. 0 means only white particals and 1 means particals with the color from the origianl image

method

Default:0

Type: number

Description: can only be chose from 0 or 1. 0 means particals move from top to botton and 1 means particals move diagonally from top-left to botton-right

path

Default:/

Type: String

Description: Path represents the url of the image which developers wants to aplly pixel rain effect on in the form of base64. Developers must convert to base64 in advance for this animation to work.
